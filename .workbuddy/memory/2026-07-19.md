# 2026-07-19

## BattleStage 场景问题诊断（不动手只给方案）

用户反馈两个问题：
1. 实际运行时 UI 整体向右偏，期望跟编辑器里看到的一致
2. 地板自动移动，希望开局静止

根因分析：
- UI 偏右：`BattleStage.tscn` 里 `BackgroundRoot/ui` 和 `ui2` 都是普通 Sprite2D，受 Camera2D 影响。
  - Warrior 全局 X = BattleRoot(-68) + UnitsRoot(148) + Warrior(195) = 275
  - 相机中心 X = 275 + look_ahead_x(150) = 425
  - UI 世界 X=640 在屏幕上渲染为 640 - 425 + 640 = 855（偏右 215 像素）
  - 解决：建一个 CanvasLayer 节点 `UILayer`，把 `ui` 和 `ui2` 移过去；CanvasLayer 下的子节点 position 是屏幕像素坐标，需要重新调整
- 地板自动移动：`scripts/stages/stage_scroll_controller.gd` 第 4 行 `scrolling_enabled: bool = true` 默认开启，_ready() 把 ground_parallax.autoscroll 设为 (-180, 0)
  - 解决：把默认值改 false，或在 BattleStage.tscn 中取消 StageScrollController 的 Scrolling Enabled 勾

关键原理：UI/HUD 必须放 CanvasLayer 下，否则会跟着 Camera2D 跑。

项目：rpg6688，Godot 4.7，主场景 `scenes/stages/chapter_01/BattleStage.tscn`。
